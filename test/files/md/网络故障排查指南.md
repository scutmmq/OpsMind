# 企业网络故障排查指南

## 一、概述

本文档面向企业 IT 运维人员和普通员工，提供常见网络故障的系统化排查方法。
遵循「从简单到复杂、从本地到远端」的排查原则，可快速定位大部分网络问题。

## 二、基础检查（5 分钟内完成）

### 2.1 物理连接

1. **网线连接**：检查网线两端指示灯是否正常亮起（绿色常亮 = 连接正常，橙色闪烁 = 数据传输中）
2. **WiFi 开关**：笔记本电脑需确认物理 WiFi 开关（通常为 Fn+F2 组合键）处于开启状态
3. **交换机端口**：检查墙壁网络端口或桌面交换机对应端口的指示灯状态

### 2.2 IP 地址获取

Windows 系统打开命令提示符（CMD），执行：
```
ipconfig /all
```

检查项：
- IPv4 地址是否以 `10.` 或 `172.16-31.` 开头（公司内网段）
- 默认网关是否正确（公司网关为 `10.0.0.1`）
- DNS 服务器是否为 `10.0.0.53` 和 `10.0.0.54`

若 IP 地址为 `169.254.x.x`，表示 DHCP 获取失败，需联系网络管理员。

### 2.3 基本连通性测试

```bash
# 测试网关连通性
ping 10.0.0.1 -n 4

# 测试 DNS 解析
nslookup www.company.com

# 测试外网连通性
ping 8.8.8.8 -n 4
```

## 三、常见场景排查

### 3.1 无法访问内网系统

**现象**：能上外网但无法访问 OA、ERP 等内网系统

**排查步骤**：
1. 确认连接的是 `Corp-WiFi` 而非 `Guest-WiFi`（访客网络仅提供外网）
2. 检查是否开启了 VPN（访问内网系统需要连接 VPN）
3. 清除浏览器缓存和 DNS 缓存：`ipconfig /flushdns`
4. 尝试使用 IP 地址直接访问（排除 DNS 问题）

### 3.2 VPN 连接失败

**现象**：VPN 客户端提示「无法连接到服务器」或「认证失败」

**排查步骤**：
1. 确认外网连接正常（访问 https://www.baidu.com 测试）
2. 检查 VPN 服务器地址：`vpn.company.com`，端口 `443`
3. 确认域账号未过期（联系 IT 服务台查询 AD 账号状态）
4. 如使用短信验证码登录，确认手机信号正常
5. 尝试切换 VPN 协议：SSTP（默认）→ L2TP/IPSec（备用）

**自助处理**：
- 重置 VPN 配置：设置 → 网络和 Internet → VPN → 删除现有连接 → 重新添加
- 下载最新 VPN 客户端：https://vpn.company.com/download

### 3.3 无线网络频繁断开

**排查步骤**：
1. 确认在 AP 覆盖范围内（通常半径 20 米无遮挡）
2. 检查是否有微波炉、蓝牙设备等 2.4GHz 干扰源
3. 更新无线网卡驱动：设备管理器 → 网络适配器 → 右键更新驱动
4. 修改电源管理设置：设备管理器 → 无线网卡属性 → 电源管理 → 取消「允许计算机关闭此设备」
5. 忘记网络后重新连接：设置 → 网络和 Internet → WiFi → 管理已知网络 → 忘记 → 重新连接

## 四、进阶排查（运维人员）

### 4.1 使用网络诊断工具

```bash
# 路由追踪
tracert target-server.company.com

# 端口连通性测试
Test-NetConnection target-server.company.com -Port 443

# 查看当前网络连接
netstat -ano | findstr ESTABLISHED
```

### 4.2 常见错误码解读

| 错误码 | 含义 | 建议操作 |
|--------|------|----------|
| DNS_PROBE_FINISHED_NXDOMAIN | DNS 解析失败 | 更换 DNS 或检查 hosts 文件 |
| ERR_CONNECTION_TIMED_OUT | 连接超时 | 检查防火墙和代理设置 |
| ERR_SSL_PROTOCOL_ERROR | SSL/TLS 错误 | 检查系统时间和证书 |
| HTTP 403 Forbidden | 无权限访问 | 确认账号权限，联系系统管理员 |
| HTTP 502 Bad Gateway | 网关错误 | 后端服务异常，联系运维团队 |
| HTTP 504 Gateway Timeout | 网关超时 | 后端服务响应慢，检查服务器负载 |
