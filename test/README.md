## 1. 环境准备

### 1.1 启动服务

在项目根目录执行：

```bash
# 启动全部服务（使用 OpenAI API 作为 LLM 后端）
docker compose up -d --build

# 或启动含本地 llama.cpp 的完整环境（需先下载模型）
docker compose --profile ai-local up -d --build
```

### 1.2 初始化数据库

```bash
docker compose exec -T postgres psql -U opsmind -d opsmind < server/migrations/init.sql
docker compose exec -T postgres psql -U opsmind -d opsmind < server/migrations/seed_essential.sql
```

### 1.3 确认服务就绪

执行 `docker compose ps`，确认所有服务状态为 healthy 或 running：

| 服务 | 预期状态 |
|---|---|
| opsmind-server | healthy |
| opsmind-web | healthy |
| postgres | healthy |
| minio | healthy |

打开浏览器访问：
- 后端健康检查：`http://localhost:8080/health` → 显示 `{"status":"ok"}`
- 前端界面：`http://localhost:3000` → 显示登录页面

## 3. 测试数据概览

测试过程中需要逐步创建以下数据。每条数据都服务于特定业务场景的验证。

### 3.1 知识库（2 个）

| 名称 | 说明 | 创建于 |
|---|---|---|
| IT 运维知识库 | 存放 VPN、打印机、邮箱、网络相关知识和文档 | Flow 3 |
| 信息安全规范库 | 存放安全制度和邮件规范文档 | Flow 3 |

### 3.2 知识文章 — 手动创建（3 篇）

| 标题 | 走哪条路径 | 验证什么 |
|---|---|---|
| VPN 连接超时问题处理 | 草稿→审核→通过→发布→停用→启用 | 完整正向生命周期 |
| Outlook 邮箱无法收发邮件排查 | 草稿→审核→**驳回**→编辑→重提→通过→发布 | 驳回后重提路径 |
| 打印机卡纸处理指南 | 保持草稿不发布 | 非法状态转换被拒绝 |

### 3.3 文档上传（6 份，4 种格式）

| 文件 | 格式 | 上传到 |
|---|---|---|
| 网络故障排查指南.md | MD | IT 运维知识库 |
| VPN使用常见问题.txt | TXT | IT 运维知识库 |
| 新员工IT入职指南.md | MD | 信息安全规范库 |
| 会议室设备操作说明.txt | TXT | 信息安全规范库 |
| 邮件系统使用规范.docx | DOCX | 信息安全规范库 |
| 信息安全管理制度.pdf | PDF | 信息安全规范库 |

### 3.4 问答测试（3 轮对话）

| 问题 | 预期 |
|---|---|
| 「如何解决 VPN 连接超时？」 | 高置信度回答，引用知识库内容 |
| 「它的备用服务器地址是什么？」（追问） | 系统理解「它」指 VPN，正确回答 |
| 「如何配置思科路由器 BGP？」 | 低置信度，引导用户提交申告 |

### 3.5 申告工单（2 份）

| 标题 | 流转路径 | 验证重点 |
|---|---|---|
| 3楼东侧打印机频繁卡纸 | 待处理→处理中→已解决 | 简单路径 + 生成知识候选 |
| VPN每隔10分钟自动断开 | 待处理→处理中→需补充→补充→处理中→已关闭 | 补充信息流程 + 站内消息 |

**操作：** 进入个人设置 → 修改密码，输入旧密码 `Admin@123`，新密码 `OpsMind@2024`。

#### 1.4 登出与重新登录

**操作：** 点击右上角头像 → 退出登录。

**系统反馈：**
- [ ] 退出成功，跳转到登录页

- [ ] 页面提示「无权限访问」或跳转回门户首页

```
## 问题描述

VPN 客户端连接时长时间无响应，最终提示「连接超时」。

## 解决方案

1. 检查当前网络是否能正常访问外网（打开百度测试）
2. 确认 VPN 服务器地址为 vpn.company.com:443
3. 尝试切换 VPN 协议：设置 → 首选项 → 协议 → L2TP/IPSec
4. 清除 VPN 客户端缓存：关闭客户端 → 删除 %LocalAppData%\AnyConnect 目录 → 重新连接
5. 尝试备用服务器 vpn2.company.com

## 如仍无法解决

请提交申告并提供当前 IP 地址（打开 https://ip.company.com 查看），以便网络组排查。
```

## 问题现象

## 排查步骤

### 第三步：尝试网页版
访问 https://mail.company.com 用相同账号登录。若网页版正常则问题在客户端。

### 第四步：重建 Outlook 配置文件
1. 关闭 Outlook
2. 控制面板 → 邮件 (Microsoft Outlook) → 显示配置文件
3. 删除当前配置文件 → 添加新配置文件
4. 按提示输入姓名、邮箱、密码

## 备注
邮箱服务器为 Exchange Online（Office 365），配置文件中服务器地址应为 outlook.office365.com。
